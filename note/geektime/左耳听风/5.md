# 左耳听风 05 day 打卡
学习《左耳听风》第 26 讲《26 | 分布式系统关键技术：流量与数据调度》。
流量调度不同于服务调度，服务调度是内部系统管理所有服务，而流量调度可以在内网，更多是在外网。服务治理属于数据中心负责，而流量调度属于边缘计算，类似 CDN。
在系统架构上应该把他们两个分开，他们属于不同层面。

### 主要功能
1.  依据系统运行情况，自动进行流量调度，无需人工干预，提升系统稳定性
2.  系统在应对流量爆发等突发情况时，弹性计算扩缩容较长时间窗口内或底层资源消耗殆尽的情况下，维护系统平稳运行。
-   服务流控。服务发现、服务路由、服务降级、服务熔断、服务保护等。
-   流量控制。负载均衡、流量分配、流量控制、异地灾备 ( 多活 ) 等。
-   流量管理。协议转换、请求校验、数据缓存、数据计算等。
### 关键技术
1.  高性能
2.  扛流量。使用集群技术，支持 Paxos、Raft、Gossip 等通信协议。
3.  业务逻辑
4.  服务化。能够通过 Admin API 不停机管理配置变更
### 状态数据调度
把有状态的服务 ( 带数据 ) 变成无状态服务，数据存储在中间件，比如 Redis/MySQL 等
### 分布式事务一致性
数据副本是分布式系统解决数据丢失异常的唯一手段。就是一份数据冗余写多份，但是会出现数据一致性问题，进而引发性能问题。以下有 4 种解决方案
-   Master-Slave 方案。
-   Master-Master 方案。
-   两阶段和三阶段提交方案。
-   Paxos 方案。
在应用层解决分布式事务问题，普遍采用两阶段提交的方案，在数据层解决的话就要用 Paxos 算法

## 总结
能解决分布式系统的数据规模问题的应该是分布式数据组件本身，它需要解决数据副本、数据一致性和分布式事务的问题，而不是把问题丢给应用层，让功能和关注点分离。

分库分表的数据库中间件TDDL/Mycat都会成为过度技术。
