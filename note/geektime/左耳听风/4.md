# 左耳听风 04 day 打卡
学习《左耳听风》第 25 讲《25 | 分布式系统关键技术：服务调度》。

服务调度包含几个关键技术点
-   服务关键程度和服务依赖关系
-   服务发现
-   整个架构的版本管理
-   服务应用生命周期全管理
### 服务关键程度和服务依赖关系
通过理解业务，划分出服务关键程度和服务之间的依赖关系，能画出服务依赖关系图。这是服务治理的前提，如果对服务不熟悉，治理就无从谈起。
划分服务关键程度就是给服务安重要程度进行分组，哪些是 L0 服务，一定不能宕机。哪些是 L1 服务，如果宕机怎么处理。哪些是 L2 服务，在系统压力非常大的情况下，可以牺牲它们保护 L0.
一定要避免服务之间有循环依赖，比如说 A 依赖 B, B 依赖 C, C 却依赖 A，这说明架构设计出现错误，导致服务之间耦合紧，服务部署非常复杂。
公司用 skywalking 做链路追踪各个服务，可能因为还没真正用起来，没发现它的优势。
### 服务状态和生命周期管理
我们通过服务注册中心管理服务的整个生命周期。
-   整个架构的服务数量
-   每个服务的版本
-   每个服务的实例数量，实例状态
-   每个服务的状态，包括服务启动，服务健康检查，服务升级，服务回滚，服务销毁等状态。
### 整个架构的版本管理
分布式架构中，需要对架构进行版本控制，管理其中各个服务的版本兼容。比如，A 服务的 1.2 版本只能和 B 服务的 2.2 版本一起工作，A 服务的上个版本 1.1 只能和 B 服务的 2.0 一起工作。这就是版本兼容性。
如果我们需要回滚一个版本，就要把该服务依赖的服务版本一起回滚。
### 资源/服务调度
服务和资源调度跟操作系统调度进程方式相似，主要涉及以下几方面。
-   服务状态的维持和拟合
-   服务的弹性伸缩和故障转移
-   作业和应用调度
-   作业工作流编排
-   服务编排
### 服务的弹性伸缩和故障转移
一个服务伸缩需要的操作步骤
-   底层资源的伸缩
-   服务的自动化部署
-   服务的健康检查
-   服务发现的注册
-   服务流量的调度
