## 高可用

极限网关的高可用部分是这样介绍的：

> 极限网关内置浮动 IP 功能，可以实现双机热备、故障转移的能力，极限网关天然提供四层网络流量的高可用，无需再额外考虑增加额外的软件和设备来保障因为停机、网络故障等造成的代理服务中断。

今天我们就来测试下极限网关的高可用功能，方案部署2个极限网关，配置浮动IP，当一个网关被kill后，另一个网关会不会正常提供访问。

### 浮动IP

每台服务器之间都必须要有 IP 才能进行通信，一台服务器的 IP 一般是提前分配好固定地址， 如果因为故障宕机，就无法对外提供服务。

浮动 IP 也叫虚拟 IP 或者动态 IP，通常是一个公开的、可以访问的 IP 地址，并且不会自动分配给实体设备。服务器管理者把该IP分配给一个或者多个实体设备。多个设备间通过内网IP进行通信，如果其中一个设备宕机，其他正常设备能继续提供服务，这样保障了服务的高可用。
