## 角色

用户关联角色，角色关联权限，一个用户可以关联多个角色，一个角色可以属于多个用户，当前角色类型有 console 和 elasticsearch

### console 角色

管理员分配给用户在 console 平台使用的角色。角色关联 console 平台权限，用户在 console 平台能够做哪些操作，查看哪些页面，点击哪些按钮，都取决于该用户关联的 console 角色权限。

#### 模拟场景

* 管理员创建 console 角色 dev，关联网关管理权限，分配该角色给用户 zhangsan。
* zhangsan 登录后只能看到网关管理模块，其他模块的业务和接口都无法操作

### elasticsearch 角色

管理员分配给用户在 elasticsearch 使用的角色。角色关联集群，索引，集群 api，索引 api。
客户端携带用户的用户名和密码发送给网关的，网关从请求中取出集群、索引，api，再从用户角色关联的权限中校验。
校验顺序：集群、索引、api

#### 模拟场景

管理员创建 elasticsearch 角色 read，关联集群 cluster1，索引 index1, 集群 api **cat**, 索引 api **index.get**
* 客户端请求 cluster2，校验不通过
* 客户端请求 cluster1 中的 index2 索引 ```GET /_cat/count/index2```，校验不通过
* 客户端请求 ```GET /_nodes```, 校验不通过
* 客户端请求 ```DELETE /index1```, 校验不通过

### 角色属性

|  字段名  |类型 | 是否必填  |备注|
|  ----  | ----  | ----  | ----  |
| id|string|必填| 主键|
|name|string|必填|角色名，不能重复|
|description|string|非必填|角色描述|
|type|string|必填|角色类型，console 和 elastisearch|
|created|time|必填|创建时间|
|updated|time|必填|修改时间|
|permission|array|必填|console 角色绑定权限列表|
|cluster|array|必填|elasticsearch 角色允许访问的集群名称，*表示所有|
|index|array|必填|elasticsearch 角色允许访问的索引名称，*表示所有|
|cluster_privilege|array|必填|elasticsearch 角色允许操作集群 api，*表示所有|
|index_privilege|array|必填|elasticsearch 角色允许操作索引 api，*表示所有|

## 操作日志

在执行数据写入或删除操作时，记录操作日志，其中包含如下属性

### 日志属性

|  字段名  |类型 | 是否必填  |备注|
|  ----  | ----  | ----  | ----  |
| id|string|必填| 主键|
|timestamp|datetime|必填|操作时间|
|metadata.labels|object|必填|业务数据|
|metadata.category|string|必填|日志分类，platform 平台，elasticsearch|
|metadata.group|string|必填|日志分组 rbac 权限|
|metadata.name|string|必填|操作模块|
|metadata.type|string|必填|操作类型 create/update/delete|
|changelog.from|string|必填|操作前的数据对象值|
|changelog.path|array|必填|数据对象名|
|changelog.to|string|必填|操作后的数据对象值|
|changelog.type|string|必填|对该数据对象的操作|
|payload|object|非必填|自定义补充数据|

| | 删除角色| 	管理员删除角色| |
| | 修改角色	| 管理员修改角色|验证前端数据的时候，把角色名重新赋值，不允许修改角色名|

||重置密码|重置用户的密码|
